# UI Audit Baseline (2026-02-09)

## Scope
- Public: `/`, `/docs`, `/pricing`, `/api-docs`
- Authenticated: `/editor`, `/profile`, `/profile/recharge`, `/profile/billing`, `/profile/billing/orders/[outTradeNo]`
- Admin: `/admin`, `/admin/users`, `/admin/analytics`, `/admin/payments`, `/admin/settings`, `/admin/settings/payment`

## Standards
- Web Interface Guidelines (Vercel Labs)
- WCAG 2.2 AA
- Mobile touch-target best practice (`>= 44x44` for key controls)

## Severity Definition
- `Critical`: blocks task completion or major legal/compliance risk
- `High`: significant accessibility/UX break for part of users
- `Medium`: usability friction or inconsistent semantics
- `Low`: polish issues, readability, consistency

## Review Matrix (Task Flows)
- Public onboarding: landing CTA -> sign-up/sign-in
- Editing workflow: upload -> select -> generate -> download
- Recharge workflow: recharge -> pay -> query status -> order detail
- Admin workflow: filter -> inspect -> action menu -> settings

## Deliverables
- Issue list with `file:line`, impact, reproduction steps, and fix recommendation
- Patch set for P0/P1 accessibility and UX issues
- Audit scripts for ongoing checks:
  - `npm run lint:a11y`
  - `npm run audit:axe`
  - `npm run audit:lighthouse`
